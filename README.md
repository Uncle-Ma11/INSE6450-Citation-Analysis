# Citation Analysis: Perfunctory vs. Non-Perfunctory Citation Identification

## Project Goal
The primary objective of this project is to develop an AI system capable of automatically distinguishing between **"Non-Perfunctory Citations"** (Meaningful, Key to the work) and **"Perfunctory Citations"** (Incidental, Background mentions) in scientific literature.

Identifying both types is equally important for our goal of creating a "weighted" citation analysis metric. We distinguish between mere acknowledgments and citations that truly influenced the methodology or theory of a paper.

---

## Dataset

### Primary Dataset: SciCite
We utilize the **SciCite** dataset (AllenAI), using the `isKeyCitation` label as ground truth for binary classification.

- **Non-Perfunctory (Meaningful)**: Mapped from `isKeyCitation=True`
- **Perfunctory (Incidental)**: Mapped from `isKeyCitation=False`
- **Data split**: Stratified 80-10-10 (train / val / test) generated by `split_dataset.py`

---

## Model Architecture

| Component | Detail |
|---|---|
| **Base Model** | `allenai/scibert_scivocab_uncased` |
| **Classifier Head** | Dropout(0.3) → Linear(768→256) → LayerNorm(256) → GELU → Dropout(0.2) → Linear(256→2) |
| **Loss Function** | Focal Loss (γ=2.0) with per-class weighting |
| **Optimizer** | AdamW, LR=3e-5, warmup_ratio=0.06, weight_decay=0.01 |
| **Input Format** | `"Section: {section}. {context}"` + `[SEP]` + cited abstract |
| **Inference Threshold** | 0.58 (calibrated on validation set) |

#### Step 1 — Stratified Dataset Resplit *(class imbalance treatment)*

> **Required before training.** The original SciCite splits suffer from **prior-shift** — the
> `isKeyCitation=True` class ratio varies significantly across the original train/dev/test files
> (≈41% train, ≈30% dev, ≈38% test). This inconsistency biases validation metrics and causes
> the model to be evaluated on a different class distribution than it was trained on, leading to
> artificially inflated or deflated F1 scores and unreliable threshold calibration.
>
> `split_dataset.py` pools the entire SciCite corpus and performs a **stratified 80-10-10
> resplit** using `sklearn.train_test_split(stratify=...)`, guaranteeing that the
> `isKeyCitation` ratio is identical across all three splits (≈41.1% Key, ≈58.9% Not-Key).
> This is the standard approach for handling class imbalance in ML research pipelines.

**Why stratified splitting matters for class imbalance:**
- Ensures val/test sets reflect the true class prior — preventing **prior-shift contamination**
- Makes threshold calibration reliable (the optimal threshold found on val generalises to test)
- Guarantees consistent Macro F1 comparisons across experiments
- Reduces variance of minority-class recall estimates in held-out sets

**Original SciCite split key-citation ratios (before resplit):**

| Split | Samples | Key% | Issue |
|---|---:|---:|---|
| `train.jsonl` | 8,194 | 41.0% | ← slightly high |
| `dev.jsonl` | 916 | 29.5% | ← **10pp lower** — severe prior-shift |
| `test.jsonl` | 1,859 | 38.4% | ← intermediate |

**After stratified resplit (80-10-10):**

| Split | Samples | Key% | Status |
|---|---:|---:|---|
| `resplit_train.jsonl` | 8,775 | 41.1% | ✅ aligned |
| `resplit_val.jsonl` | 1,097 | 41.1% | ✅ aligned |
| `resplit_test.jsonl` | 1,097 | 41.1% | ✅ aligned |

```bash
python scripts/split_dataset.py
```

- **Requires:** All three original files in `data/raw/scicite/scicite/` (from Step 0)
- **Output:** `data/raw/scicite/scicite/resplit_{train,val,test}.jsonl`
- **Skip if:** `resplit_train.jsonl` already exists in that directory

**All CLI options:**

| Option | Default | Description |
|---|---|---|
| `--ratio TRAIN VAL TEST` | `0.8 0.1 0.1` | Split proportions (must sum to 1.0) |
| `--seed` | `42` | Random seed for reproducibility |
| `--data_dir` | `data/raw/scicite/scicite` | Directory with original JSONL files |
| `--out_dir` | *(same as `--data_dir`)* | Output directory for resplit files |
| `--prefix` | `resplit` | Filename prefix (`resplit_train.jsonl`, etc.) |

**Example — alternate 70-15-15 split:**
```bash
python scripts/split_dataset.py --ratio 0.7 0.15 0.15 --seed 42
```

**Pass resplit files explicitly to the training script:**
```bash
python scripts/train_scicite_model.py \
    --train_file data/raw/scicite/scicite/resplit_train.jsonl \
    --dev_file   data/raw/scicite/scicite/resplit_val.jsonl \
    --test_file  data/raw/scicite/scicite/resplit_test.jsonl
```

> **Note:** The default `train_scicite_model.py` automatically detects and uses the resplit
> files when they are present in `data_dir`, so explicit `--*_file` flags are only needed
> if you changed `--out_dir` or `--prefix`.

### Milestone 2 — Model Training & Inference

#### Step 3 — Train the Model
> **Required before any evaluation or inference step.**
> Trains SciBERT with the optimal hyperparameter configuration found by grid search.
> By default, uses cited-paper abstract enrichment when `abstracts_mapping.json` is present.

**Standard training (with abstract enrichment):**
```bash
python scripts/train_scicite_model.py
```

**Ablation — context only, no abstract enrichment:**
```bash
python scripts/train_scicite_model.py --no_abstracts
```

- **Requires:** Steps 0–1 (Step 2 strongly recommended for the default mode)
- **Output:**
  - `best_model.pt` — best checkpoint by Val Macro F1
  - `results/training_metrics.json` — efficiency metrics (time, VRAM, FLOPs)
  - `results/learning_curve.png` — loss + F1 curves

**Input modes:**

| Mode | Flag | Input to SciBERT |
|---|---|---|
| With abstracts *(default)* | *(none)* | `"Section: {s}. {context}"` `[SEP]` `{cited_abstract}` |
| Context only | `--no_abstracts` | `"Section: {s}. {context}"` only |

> **When to use `--no_abstracts`:**
> - Abstracts have not been fetched yet (`fetch_abstracts.py` still running)
> - Running an ablation study to measure the contribution of abstract enrichment
> - The `abstracts_mapping.json` file is missing and you want to train immediately

**Key CLI options:**

| Flag | Default | Description |
|---|---|---|
| `--no_abstracts` | `False` | Disable abstract enrichment (context-only input) |
| `--epochs` | `8` | Number of training epochs |
| `--lr` | `3e-5` | Learning rate (optimal from sweep) |
| `--focal_gamma` | `2.0` | Focal loss γ — higher = more focus on hard/minority samples |
| `--batch_size` | `16` | Batch size per step |
| `--accum_steps` | `2` | Gradient accumulation steps (effective batch = 32) |
| `--patience` | `3` | Early stopping patience (epochs without improvement) |
| `--seed` | `42` | Random seed |
| `--debug` | `False` | Tiny 64-sample run to verify the pipeline quickly |

**Optional — Hyperparameter Sweep:**
```bash
# Sweep with abstracts
python scripts/train_scicite_model.py --sweep

# Sweep without abstracts (ablation)
python scripts/train_scicite_model.py --sweep --no_abstracts
```
- Runs 24 configurations from the grid (LR × gamma × warmup_ratio × label_smoothing).
- **Output:** `results/sweep_results.json` — full leaderboard
